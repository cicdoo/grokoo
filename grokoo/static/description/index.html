Grokoo

Chat with your Odoo — safely.

An in-Odoo AI assistant powered by the xAI Grok CLI. It answers questions, queries your data, and builds reports — running as the logged-in user, never as superuser, with a full audit trail.

Security first — by design

🔒 Runs as the user

Every ORM action executes with su=False. The user's access rights and record rules are enforced automatically — the model can never see or do more than the person using it.

🧮 Read-only SQL, really

sql_select is double-guarded: a SELECT-only validator plus Postgres SET TRANSACTION READ ONLY and a statement timeout, gated to a dedicated AI SQL Analyst group.

🧾 Full audit log

Every tool call — model, method, SQL, row counts, outcome, even denials — is written to an immutable log on a separate committed cursor, so it survives rollbacks.

🔑 Per-user credentials

Each user connects with their own credentials — importing their Grok login (so their own xAI subscription applies) or an xAI API key. No shared key required; credentials are stored per user in a private .grok dir (mode 0600), never in the database.

Sandboxed. No surprises.

Grok runs headless with its built-in shell/write/network tools withheld, so it can't reach the shell, filesystem, or network on its own. The model acts only through a small set of Odoo tools over a loopback bridge that holds no database credentials — just a short-lived, session-scoped capability token. Replies and tool activity stream into the chat in real time over the Odoo bus.

What it can do

  • Answer questions about your records in natural language
  • Search, read, and group data with the user's permissions
  • Run read-only SQL reports (for the SQL Analyst group)
  • Stream answers and tool steps live into the chat
  • Opt-in, per-user write tools (create / write / unlink / actions)
  • Zero-trust mode to strip writes globally or per user
  • Manager audit log of every tool call and denial
  • Per-user Grok credentials — no shared key
